# Genome Decoder — расшифровка генома за 14 шагов

Локальный пайплайн на чистом Python (+ опц. R для HLA). Берёт сырой файл генотипирования
или WGS-VCF и собирает клинически-осмысленный отчёт. Принципы: Python-детерминизм для механики, AI только для интерпретации, каждый вывод —
с проверяемым источником (ClinVar/gnomAD), мульти-агентное ревью для надёжности.

## ⚠️ Перед стартом — ОБЯЗАТЕЛЬНО уточни у пользователя

1. **Какой файл генома?** Поддержка:
   - 23andMe / AncestryDNA raw `.txt` (чип, ~600К SNP, build 37) — ✅ работает сразу
   - Genotek / любой VCF (`.vcf`/`.vcf.gz`) — ✅
   - Полный WGS VCF (Dante Labs/Nebula/Sequencing.com) — ✅ (больше покрытие)
   - ⚠️ Спроси **build**: пайплайн настроен на **GRCh37/hg19** (как у raw-чипов). Если данные
     GRCh38 — нужен лифтовер ИЛИ замена URL баз на GRCh38.
2. **Где файл лежит?** (абсолютный путь).
3. **Что интересует?** (носительство для планирования семьи / лекарства / риски / питание /
   происхождение / HLA — всё или конкретное). Это влияет на акцент в разборе.
4. **Делал ли импутацию?** Если нет — предложи (даёт ×10–50 покрытия, см. ниже). Без неё
   ClinVar/носительство идут по типизированным SNP (меньше находок, но работает).
5. **Приватность:** геном — чувствительные данные. Всё считается ЛОКАЛЬНО; в облако уходят
   только: импутация (опц., Michigan) и точечные API-запросы (gnomAD/VEP/CADD по конкретным
   позициям). Предупреди пользователя; ничего не публикуй без согласия.

## Установка (один раз)

```bash
cd genome-decoder
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt        # pysam (для импутации), всё остальное — stdlib
# опц. для полного HLA: R + Bioconductor HIBAG + модель (см. «HLA» ниже)
```

## Запуск (базовый — работает на любом чипе/VCF сразу)

```bash
export GENOME_INPUT="/полный/путь/к/геному.txt"   # или .vcf/.vcf.gz
export SAMPLE_ID="me"
cd steps && python run_pipeline.py
```

Оркестратор сам пройдёт шаги, скачает ClinVar (~200МБ в `.cache/`) и соберёт отчёт в
`out/genome-report-<дата>.md`. Шаги, требующие сети (gnomAD/VEP/CADD API), и тяжёлые
(AlphaMissense 622МБ) — авто-скачиваются/пропускаются с понятным сообщением.

## 14 шагов — что делает, что нужно

| # | Шаг | Что делает | Реализация |
|---|-----|-----------|-----------|
| 1 | Нормализация | raw → единый TSV (авто-детект 23andMe/VCF) | `step1_normalize.py` |
| 2 | Импутация | дорисовка генотипов по статистике популяций (×10–50) | Michigan Imputation Server (облако, бесплатно) — см. ниже |
| 3 | Аннотация | эффект варианта (missense/стоп/сплайс) | `step3_vep.py` (Ensembl VEP REST по значимым) |
| 4 | ClinVar | сверка с мировой базой Pathogenic→Benign | `step4_clinvar.py` (качает ClinVar GRCh37) |
| 5 | gnomAD-фильтр | частый вариант ≥5% → не патология (BA1) | `step5_filter.py` + `step5b_gnomad.py` (API popmax) |
| 6 | GWAS+PRS | полигенные шкалы риска (перцентиль) | `step6_prs.py` (PGS Catalog) |
| 7 | Фармакогеномика | как метаболизирует лекарства (фенотипы) | `step7_pharmacogenomics.py` (CPIC-таблица) |
| 8 | AI-скоры | предсказание вредности неизученных вариантов | `step8_alphamissense.py` + `step8b_cadd_spliceai.py` |
| 9 | Трейты | мышцы/хронотип/стресс/пигментация | `step9_traits.py` |
| 10 | Нутригеномика | усвоение еды (лактоза/кофеин/фолат/…) | `step10_nutrigenomics.py` |
| 11 | Происхождение | европ./азиат./афр. AIM + mtDNA-профиль | `step11_ancestry.py` |
| 12 | HLA | drug-safety + целиакия (тег-SNP) + полное (HIBAG) | `step12_hla.py` + `hla_hibag.R` |
| 13 | Носительство | рецессивные болезни для планирования семьи | `step13_carrier.py` (ACMG SF) |
| 14 | Сборка | единый отчёт | `report.py` |

## Импутация (шаг 2, опц., но даёт главный прирост покрытия)

1. `cd steps && python michigan_prep.py` — нарежет VCF по хромосомам в `.cache/michigan-input/`
   (с обходом требования Michigan «≥5 образцов» через дублирование).
2. Пользователь заливает их на https://imputationserver.sph.umich.edu (бесплатный аккаунт):
   панель **HRC r1.1 2016**, build **GRCh37**, phasing Eagle, rsq off, Mode QC&Imputation.
3. Результат `chr*.dose.vcf.gz` (запаролен) распаковать в `.cache/imputed/`.
4. Перезапустить `python run_pipeline.py` — шаги 4/5/8/13 пойдут на импутированных.

## Полное HLA (шаг 12b, опц.)

```bash
brew install r                      # или другой способ установить R
```

## Что в результате

`out/genome-report-<дата>.md` — разделы: фармакогеномика (карточка врачу), носительство,
gnomAD-отсев шума, PRS-перцентили, трейты, нутригеномика, AlphaMissense/CADD, VEP, HLA,
происхождение. Плюс сырые TSV по шагам в `out/`.

После прогона — **сделай персональный разбор**: приоритизируй находки (носительство для
планирования семьи > фарма/HLA > PRS > одиночные нутри/трейт-SNP), свяжи с образом жизни
пользователя, дай 3–7 конкретных действий. Всегда повторяй: это скрининг, не диагноз.

## Честные ограничения (обязательно проговори пользователю)

- **Сырые данные чипа ложно-завышают редкие патогенные** (Tandy-Connor 2018, ~40% FP).
  Пайплайн отсекает: инделы (чип их не типирует), частые ≥5% (BA1), гомозиготные артефакты
  (референс=минорный аллель). Но любую находку подтверждать клинической лабораторией.
- **Импутация вероятностна** (порог R²); обход «≥5 образцов» дублями делает R² консервативным.
- **Не смешивать уровни доказательности:** ClinVar+gnomAD > PGx-теги/HLA > PRS > AlphaMissense/нутри.
  AlphaMissense high-score = «predicted deleterious», НЕ клиническая патогенность.
- **PRS = относительный риск** (перцентиль в популяции), не вероятность болезни; частоты европейские.
- **Это научпоп, не медицина.** Обязательна консультация с врачом/генетиком. Авторы не несут
  ответственности за решения по здоровью на основе этого инструмента.

## Надёжность через мульти-модельное ревью (рекомендуется)

Мульти-модельное ревью (несколько кругов разными моделями): после прогона прогони отчёт через
2-ю модель (напр. `codex exec "критически разбери эти геном-находки…"`) и через
самокритику — ловит раздутые формулировки и методологические ошибки. В разработке этого
пайплайна 5 таких кругов поймали: индел-артефакты, нужен gnomAD popmax, HLA-QC по R².
